Storing Data

When storing data in RAP, it's important to take into account that multiple users and RAP applications may share the same Java VM and the same file system. To address this, RAP provides basic, out-of-the box storage solutions for three common cases. Of course, these are not suited for the permanent storage of huge or sensitive data structures.

Interface Access Scope stores cleared
ApplicationContext RWT.getApplicationContext(),
from any thread 		application any object when application is stopped/restarted
UISession RWT.getUISession(),
requires session context 		UI Session any object when session expires
SettingStore RWT.getSettingStore(),
requires session context 		user (persistent) strings never, cookie lasts 3 month by default

All these Data Stores have at least the three methods setAttribute, getAttribute and removeAttribute. Note that the settings store relies on cookies, which may be deleted or disabled by the user. It is also possible to identify the user by any other means and use the method loadById to initialize the SettingStore.

If you need to use the UISession or SettingStore from from a background thread, read Session Access from background thread.
